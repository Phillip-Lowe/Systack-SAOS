## 1. Overview

Automated outreach sequencer that sends timed follow-up messages to leads generated by the Lead Scraper. Multi-step sequences with conditional branching based on response.

**Workflow:** `Green_Systems_—_Outreach_Sequencer.json`

---

## 2. System Architecture

```
Lead enters sequencer (status = 'new')
  → Step 1 (Day 0): Initial outreach email/DM
    → Response? → Move to 'responded', branch to human follow-up
    → No response?
      → Step 2 (Day 3): Follow-up #1 — different angle
        → Response? → Human follow-up
        → No response?
          → Step 3 (Day 7): Follow-up #2 — value add
            → Response? → Human follow-up
            → No response?
              → Step 4 (Day 14): Breakup email
                → No response → Status: 'dead'
```

## 3. Sequence Templates

### Food Truck Pitch Sequence

| Step | Day | Message Type |
|------|-----|-------------|
| 1 | 0 | Intro: "We build online ordering for food trucks — here's an example" |
| 2 | 3 | Value: "This truck increased orders 30% with our system" |
| 3 | 7 | Urgency: "Summer season is starting — ready to take online orders?" |
| 4 | 14 | Breakup: "If the timing isn't right, we'll check back next season" |

### Service Business Sequence

| Step | Day | Message Type |
|------|-----|-------------|
| 1 | 0 | Intro: "Automated booking + no-show prevention for [industry]" |
| 2 | 3 | Case study: "Here's how [similar business] reduced no-shows 80%" |
| 3 | 7 | ROI: "Average client saves 10 hrs/week + $500/mo" |
| 4 | 14 | Breakup: "Here's our info if you ever want to explore this" |

---

## 4. Configuration

| Variable | Purpose |
|----------|---------|
| `SEQUENCE_TYPE` | Which template to use |
| `OUTREACH_CHANNEL` | Email, DM, or both |
| `RESPONSE_WINDOW_DAYS` | Days to wait between steps |

---

## 5. Lead Status Flow

```
new → contacted (Step 1 sent)
  → responded (reply received) → qualified → client
  → dead (no response after all steps)
```
