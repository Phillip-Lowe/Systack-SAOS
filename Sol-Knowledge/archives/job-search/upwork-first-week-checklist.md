## Response Templates (For Client Replies)

### Client: "Tell me more about your experience"
```
Great question. I've built automation systems across industries — logistics, SaaS, service businesses. 

My signature project is Systack.net, a platform I built using n8n to automate lead generation, scoring, and outreach. It runs 24/7 with zero manual intervention.

I also built a multi-agent AI system that coordinates 6 specialized agents for business operations — coding, design, validation, communication.

The common thread: I don't just connect apps. I design systems that think ahead, handle errors, and scale with your business.

Want to jump on a quick call? I can screenshare some examples.
```

### Client: "Can you do this cheaper?"
```
I understand budget is a factor. A few options:

1. **Automation Audit ($250)** — I analyze your process, deliver a roadmap. You can execute it yourself or hire me later.

2. **Scope reduction** — We start with the core workflow ($500 package), add complexity later.

3. **My rate stays at $75/hr** — but I'm efficient. Most projects finish faster than estimated because I've built similar systems before.

What matters most: getting it right the first time. Cheap automation that breaks costs more than doing it manually.

What would work best for you?
```

### Client: "How soon can you start?"
```
I can start immediately. Typical turnaround:

• Simple workflow (1-2 apps): 3–5 days
• Medium system (3-5 apps, logic): 1–2 weeks  
• Complex build (multi-workflow, monitoring): 2–3 weeks

I deliver updates every 2–3 days so you're never guessing where things stand.

Ready when you are.
```

### Client: "Have you worked with [specific tool/industry]?"
```
[I have / I haven't] worked with [tool/industry] specifically, but here's why that doesn't matter:

I've integrated 20+ different tools across CRMs, databases, messaging platforms, and APIs. The patterns are the same — data in, logic applied, action out. What changes is the API documentation, and I'm fast at reading those.

My 12 years in operations means I understand the business side: what you're trying to accomplish, not just the technical connection. That's where most automation projects fail — they work technically but don't solve the real problem.

Want to tell me more about what you're trying to achieve? I can confirm fit in 5 minutes.
```

## Troubleshooting: If Nothing's Working After Week 1

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| No profile views | Profile not optimized for search | Add more keywords: "n8n", "automation", "workflow" |
| Views but no messages | Description doesn't hook | Lead with client pain point, not your credentials |
| Messages but no calls | Not proposing next step | Always end with "Want to jump on a quick call?" |
| Calls but no offers | Price too high OR trust too low | Lead with audit ($250) to build relationship |
| Offers but too low | Competing on price, not value | Emphasize outcomes, not hours. Case studies help. |
