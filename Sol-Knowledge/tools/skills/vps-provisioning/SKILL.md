# VPS Provisioning

Provision cloud VPS instances for SAOS client deployments. Complete pipeline: create VPS → install software → configure Tailscale → deploy n8n templates → send branded welcome email.

## When to Use
- New SAOS client signs up and needs their own agent infrastructure
- Scaling existing client to higher tier (more resources)
- Replacing failed or compromised VPS
- Demo environment for prospect evaluation

## Tiers

| Tier | Specs | Monthly Cost | Use Case |
|------|-------|------------|----------|
| Business | 4 vCPU, 16GB RAM, Ubuntu 22.04 | $96/mo | Single business, ~5 users |
| Enterprise | 4 vCPU, 24GB RAM, Ubuntu 22.04 | $192/mo | Multiple locations, 20+ users |

## Pipeline

1. **Create VPS** — Vultr API v2, cloud-init, tier-based plan selection
2. **Install Stack** — Docker, n8n, PostgreSQL, Ollama, Tailscale
3. **Configure Tailscale** — Tagged device for unlimited clients (free tier)
4. **Deploy Templates** — Import n8n workflows per tier (order system, booking, invoice parser)
5. **Health Check** — Port/service validation before delivery
6. **Send Email** — Branded HTML welcome with Tailscale URL, n8n URL, credentials

## Requirements

- **Vultr API key** — `credentials/Green/Vultr/VULTR API`
- **Tailscale auth key** — For client VPS to join tailnet
- **Tailscale API key** — For management operations
- **n8n API key** — For workflow deployment

## Quick Start

```bash
# Provision business tier
python3 saos/scripts/provision_vps.py --client-id 001 --tier business --email "client@example.com"

# Full pipeline (VPS → Templates → Identity → Health → Email)
python3 saos/scripts/provision_pipeline.py --client-id 001 --tier business --email "client@example.com"

# Test provisioning (dry-run, cheapest instance)
python3 saos/scripts/provision_vps.py --client-id TEST --tier test --email "test@systack.net"
```

## Tailscale Multi-Client

Free Tailscale tier supports unlimited **tagged devices**:

```python
# Join client's VPS to SyStack tailnet
tailscale up --authkey ${TAILSCALE_AUTH_KEY} --advertise-tags tag:client-001

# Access from anywhere
https://client-001.systack-vpn.ts.net/
```

## Client Identity Generation

Each client workspace needs:
- `AGENTS.md` — Custom rules
- `MEMORY.md` — Empty template
- `SOUL.md` — Personality config
- `TOOLS.md` — Tool configs
- `USER.md` — About the business

Generated by: `saos/scripts/generate-identity.py`

## Health Checks

Before marking "delivered":
- [ ] n8n responds on port 5678
- [ ] Ollama responds on port 11434
- [ ] Postgres responds on port 5432
- [ ] Tailscale URL resolves via HTTPS
- [ ] Webhook test succeeds

## Credentials Storage

Client credentials stored in:
`credentials/<client-id>/<service>/<credential-name>`

Never hardcode in scripts — use Python file I/O to read.

## Security Rules

- Tailscale only (no exposed ports)
- Agent runs on VPS, not client computer
- No passwords in config files
- Local-only AI models by default (cloud opt-in)

## Post-Provisioning

1. Add client to `clients/<id>/` directory
2. Record deployment in `provisioning-log.json`
3. Schedule 30-day health check
4. Add to n8n monitor alerts

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Vultr API timeout | Retry with exponential backoff |
| Tailscale auth expired | Generate new auth key |
| n8n import fails | Check workflow JSON compatibility |
| Ollama model download fails | Use smaller model (qwen2.5:7b ≈ 4.4GB) |
